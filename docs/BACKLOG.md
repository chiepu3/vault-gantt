## 0. 🔴 実用ブロッカー（最優先・順序無視で修正）

ユーザー実機確認で「サブタスク関連が一切動かない」と報告された事象の根本原因群。

| # | 問題 | 根本原因 | 対処 |
|---|---|---|---|
| B1 | Ganttで親タスク名が横スクロールで消える | `.vg-gantt-row-left` に `position: sticky; left: 0` がない（header-leftのみ修正済み・row-left未適用） | CSS 1行追加 |
| B2 | **サブタスクを作ってもGanttに永遠にバーが出ない** | `plannedStartDate`/`plannedEndDate` を設定するUIが**プラグイン全体に存在しない**。Ganttは日付なしサブタスクを描画対象から除外（gantt-layout-engine.ts:18）するため、新規サブタスクは必ず不可視 | 旧プラグインでの主経路は「空セル右クリック→その日にサブタスク作成」(M4)と「ポップオーバーの日付入力」(M4)。**M4の空セルメニュー+ポップオーバーを前倒しで最小実装**するか、暫定でWorkbenchに予定開始/終了列を追加 |
| B3 | Workbenchのサブタスク行が表示専用 | サブタスク行は完了チェックボックス以外すべて読み取り専用（WorkbenchTable.svelte:303-318）。title/status/期限/タグ/Current Statusの編集経路なし | サブタスク行にもdblclickインライン編集を実装（親行と同じ仕組みの拡張） |
| B4 | 旧Vaultからの移行ができない | `migrateLegacyTaskNote()` はdomain層に実装済みだが**コマンドとして未配線**（main.tsのコマンドは4種のみ）。受け入れ基準1「移行コマンドで無損失変換」が未達 | `migrate-legacy-notes` コマンド追加（対象スキャン→プレビュー→実行） |
| B5 | 自動優先度が経時更新されない | priority再計算は書き込み時のみ（core-task-api.ts:121）。旧版はonloadで毎日再計算。放置タスクの期限接近が優先度に反映されない | 起動時+日付変化時の再計算ジョブ、または表示時算出に変更 |

## 推奨着手順

1. **B1〜B5（ブロッカー）**: sticky CSS → サブタスク日付付与の最小経路（空セルメニュー or ポップオーバー前倒し）→ サブタスク行編集 → 移行コマンド → 自動優先度
2. **M4本体**: ポップオーバー→空セルメニュー→右クリックメニュー→マーカー→親期限★→親行操作→一括移動→タグ→インライン編集
3. **祝日基盤の前倒し**（M5から）: 営業日スナップが嘘をつかないよう、祝日3層+設定UIはM4と並行実装を推奨
4. **M2残**: 検索範囲拡大・絞込UI・フラット表示・task-finder
5. **M5残 → M6**

> 本台帳は実装完了のたびに ✅ へ更新し、「現行機能同等」（M4完了条件）・「機能フル同等」（M5完了条件）の判定に使う。
